## TL;DR
 
- **Self-hosted LobeChat saves $295,000-$430,000 over 3 years** versus SaaS AI alternatives at 200 users, while offering capabilities neither vendor can match: custom MCP tooling, EU data residency, model choice.
- **We are GDPR-bound to self-host** if we handle client personal data in our AI workflows. Sending M&A counterparty data to US-based inference infrastructure exposes us to regulatory enforcement and client contract breaches.
- **The reversal trigger is narrow:** if a compliant, on-premises deployable AI assistant reaches market at under $15/user/month with equivalent functionality, the build argument collapses. No such product exists today.

## 1. The Question
 
The CTO asks why we are self-hosting LobeChat instead of purchasing ChatGPT Enterprise or Microsoft Copilot. This memo defends the self-hosted choice for a 200-user company operating in a regulated sector (legal services), but states explicitly the conditions under which the answer changes.

## 2. Three-Year TCO Comparison (200 users)
 
### Self-Hosted LobeChat (this stack)
 
From Q3, at 200 MAU the monthly cost is approximately:
- Fixed infra: $600/month
- Variable tokens (200 users x 440 msg x 1,300 tokens x blended $0.925/$5.375/MTok): ~$540/month
- **Total: ~$1,140/month**
| Cost component | 3-year total |
|---|---|

### ChatGPT Enterprise
 
- List price: $60/user/month (OpenAI enterprise pricing, 2025)
- 200 users x $60 x 36 months = **$432,000**
- No implementation cost (SaaS)
- No maintenance (vendor-managed)
- **ChatGPT Enterprise 3-year TCO: $432,000**
### Microsoft Copilot (M365 Copilot)
 
- List price: $30/user/month (Microsoft 365 Copilot, 2025)
- Assumes M365 E3 already licensed (otherwise add $22/user/month)
- 200 users x $30 x 36 months = **$216,000**
- Integration cost (connecting to internal systems via Graph API connectors): ~$20,000
- **Copilot 3-year TCO: $236,000**
### Summary
 
| Option | 3-year TCO | vs. Self-hosted |

Self-hosted is the least expensive option over 3 years by a significant margin. The crossover point where Copilot becomes cheaper than self-hosted requires the DevOps maintenance cost to exceed $3,667/month, meaning a dedicated half-time engineer. That would indicate an unusually complex deployment.

## 3. Lock-in Analysis
 
| Dimension | Self-hosted LobeChat | ChatGPT Enterprise | Microsoft Copilot |
|---|---|---|---|
| **Data lock-in** | None. PostgreSQL data portable via `pg_dump`. S3/MinIO objects exportable. Qdrant vectors re-embeddable from source. | High. Conversation history in OpenAI's infrastructure; export API exists but format is non-standard JSON. | High. Data in Microsoft 365 tenant; export requires admin tooling and is incomplete for AI interactions. |
| **Identity lock-in** | None. Casdoor federates with any OIDC/LDAP provider. | Medium. Tied to OpenAI accounts; SSO via SAML requires Enterprise tier. | High. Tied to Microsoft Entra ID; migrating away requires full identity migration. |
| **Model lock-in** | None. OpenRouter gives access to 200+ models; swap in `docker-compose.yml` env var. | High. GPT-4o only; no option to run a different model in the same interface. | High. GPT-4 via Azure OpenAI only; no model choice. |
| **Tooling lock-in** | None. MCP protocol is open standard; tools are containerised and re-deployable. | Medium. Custom Actions are proprietary; migrating requires rewriting integrations. | High. Copilot connectors are proprietary Microsoft Graph extensions with no portability. |
| **Exit cost** | Low. 2 weeks to migrate data + re-point DNS. | High. Data export + re-training users + rebuilding integrations: 6-8 weeks estimated. | Very high. Tenant migration + connector rewrite + identity migration: 3-6 months estimated. |
 
The non-obvious lock-in risk in SaaS AI tools is model lock-in. When OpenAI or Microsoft deprecates a model version (as has happened with GPT-3.5, GPT-4, GPT-4 Turbo), all integrations built against that model's output format break simultaneously, with no ability to pin the version. Self-hosted operators control the upgrade cadence.

## 4. Data Sovereignty and GDPR Compliance
 
For a legal services firm handling M&A counterparty data, client financial records, and employee personal data, GDPR Article 44 prohibits transfers of personal data to third countries without adequate safeguards. Both ChatGPT Enterprise and Copilot send inference requests to US-based data centres.
 
OpenAI's Data Processing Agreement (March 2025) offers Standard Contractual Clauses for EU customers, but SCCs do not eliminate US government access obligations under FISA Section 702 - a fact noted explicitly by the European Data Protection Board in its July 2023 opinion. The EU-US Data Privacy Framework (DPF, effective July 2023) provides a transfer mechanism, but its legal stability is uncertain after the Schrems I and II precedents.
 
Self-hosted on AWS eu-west-1 (Ireland): inference via OpenRouter's EU-routed endpoints, data at rest on AWS EC2/RDS/S3 within the EU, no cross-border personal data transfer. For our specific use case, this is not a preference - it is a compliance requirement. The DPO's Q1 2025 halt of the ChatGPT Enterprise pilot is on record.
 
**Sector-specific constraint (legal):** Bar association rules in Spain (Estatuto General de la Abogacía, Art. 21) impose perpetual client confidentiality obligations. A data breach caused by transmitting client matter data to a third-party US AI provider would constitute a disciplinary infraction regardless of technical safeguards. Self-hosted architecture eliminates this exposure at the structural level.

## 5. Capability Gap
 
### What Self-hosted Does That SaaS Does Not
 
| Capability | Self-hosted LobeChat | ChatGPT Enterprise | Copilot |
|---|---|---|---|
| Custom MCP tools (Notion, S3 archive, filesystem grep) | Yes - native MCPHub integration | No - limited to OpenAI Actions (external HTTP only) | No - Microsoft Graph connectors only |
| Custom RAG over internal documents | Yes - Qdrant + text-embedding-3-small, full control over chunking and retrieval | Limited - file upload to ChatGPT is ephemeral; no persistent vector store | Limited - Semantic Index, not configurable |
| Model selection per query | Yes - route tasks to Haiku (cheap) or Sonnet (capable) per token budget | No - GPT-4o only | No - GPT-4 only |
| EU data residency guarantee | Yes - all data on AWS eu-west-1 | No | No (EU tenant option exists for Copilot but is not the default) |
| On-premises / air-gapped deployment | Yes - swap OpenRouter for vLLM on local GPU | No | No |
 
### What SaaS Does That Self-hosted Does Not
 
| Capability | Self-hosted LobeChat | ChatGPT Enterprise | Copilot |
|---|---|---|---|
| Uptime SLA | None (best-effort) | 99.9% contractual SLA | 99.9% via M365 SLA |
| Vendor support | Community (GitHub Issues) | Enterprise support with named TAM | Microsoft Premier Support |
| Mobile applications | Web-only | iOS/Android apps | iOS/Android + Teams integration |
| Model refresh cadence | Manual (operator updates docker-compose.yml) | Automatic | Automatic |
| Compliance certifications | None (AWS infrastructure certs only) | SOC 2, ISO 27001, FedRAMP | SOC 2, ISO 27001, FedRAMP, HIPAA |
 
The uptime SLA gap is the most material operational risk. A 0.1% downtime difference (99.9% SLA vs. 99.5% observed self-hosted) is 4.4 extra hours of downtime per year. For a legal team where AI-drafted memos are expected before a 10:00 AM partner call, a 30-minute outage has real productivity and reputational cost. The Multi-AZ RDS + EC2 instance recovery configuration in the prod architecture mitigates this, but does not eliminate it.

## 6. Recommendation
 
**Self-host LobeChat if all of the following are true:**
 
1. The organisation operates in a regulated sector (legal, financial, healthcare) where client data sovereignty is a compliance requirement, not a preference.
2. The team has at least 0.2 FTE DevOps capacity to operate the stack: patching, scaling, incident response.
3. The use case requires custom MCP tool integrations or custom RAG over internal documents that cannot be replicated via SaaS connectors without introducing additional data controllers.
4. User count is at or above 50, making the per-user economics competitive vs. SaaS (break-even is around 85 MAU per Q3).
**Otherwise buy - Copilot recommended over ChatGPT Enterprise - if:**
 
- The team has no DevOps capacity and needs a zero-ops AI assistant.
- The organisation's data classification allows transmission to US-based infrastructure.
- The primary use case is Microsoft 365 document automation (Word, Excel, Teams), where Copilot's native integration provides value LobeChat cannot replicate.
- User count is below 30, where the $600/month infra floor makes self-hosted uneconomical.
---
 
## 7. Reversal Trigger
 
The recommendation flips to "buy" if either of the following occurs:
 
1. **A deployable compliant AI assistant reaches under $15/user/month with equivalent capability.** At that price the self-hosted TCO advantage disappears and the operational overhead is not justified. No such product exists as of May 2026.
2. **The production AWS account loses DevOps support.** This stack requires human intervention for security patching, model updates, and incident response. An unmaintained self-hosted AI platform becomes a liability - both operationally (outages) and legally (unpatched CVEs in a data-handling system). In that scenario Copilot is the correct fallback: the data residency risk is lower than the risk of an unmaintained internet-facing system.
